# 智慧农业多智能体协作框架说明

本项目为智慧农业多智能体协作通用框架，采用模块化和解耦设计，支持多种农业场景（如灌溉、施肥、病虫害防治等），便于后续扩展和维护。项目结构清晰，功能分区明确，方便团队协作和后续开发。

## 目录结构与功能简介

- **main.py**  
  项目主入口，负责系统的整体流程控制和用户交互。运行此文件可体验完整的智能体协作流程。

- **agents/**  
  智能体相关核心代码，包含各类智能体及调度、权限管理等模块。
  - `base.py`：智能体基类，定义统一接口规范。
  - `sensor.py`：数据采集智能体，采集环境数据并支持异常检测。
  - `analysis.py`：分析智能体，分析环境数据并生成决策建议。
  - `irrigation.py`：执行智能体，控制灌溉设备并反馈状态。
  - `permission.py`：权限管理模块，负责智能体权限注册与校验。
  - `orchestrator.py`：调度智能体，负责任务编排、状态监控和异常处理。
  - `__init__.py`：包初始化文件。
  - `__pycache__/`：Python自动生成的缓存文件夹。

- **data/**  
  数据处理相关代码。
  - `data_loader.py`：数据加载与处理工具。
  - `__init__.py`：包初始化文件。

- **util/**  
  工具类和辅助功能代码。
  - `logger.py`：日志管理工具，统一日志输出。
  - `__init__.py`：包初始化文件。

- **tests/**  
  单元测试代码，保障各模块功能正确性。
  - `test_agents.py`：智能体相关测试用例。
  - `__init__.py`：包初始化文件。

## 使用建议

- 新增智能体时，建议在 `agents/` 目录下新建文件并继承 `base.py` 中的基类，实现对应业务逻辑。
- 所有智能体的权限需在 `permission.py` 中注册和校验，确保安全合规。
- 通过 `orchestrator.py` 实现多智能体的任务流编排和扩展，支持多场景协作。
- 数据相关功能建议放在 `data/` 目录，工具类放在 `util/` 目录，测试代码放在 `tests/` 目录，保持结构清晰。
- 运行 `main.py` 可体验完整的智能体协作流程。
